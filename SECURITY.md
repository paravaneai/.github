# Security Policy

Paravane Labs takes security reports seriously.

## Reporting a vulnerability

Please report suspected vulnerabilities by email:

```text
security@paravane.io
```

Do not open a public GitHub issue for a vulnerability, exposed credential, bypass, abuse path, or sensitive operational weakness.

## What to include

A useful report includes:

- The affected repository, product, endpoint, or workflow.
- A concise description of the issue.
- Steps to reproduce or a proof of concept, when safe to share.
- The security impact and any known limitations.
- Whether the issue may involve exposed data, credentials, authentication, authorization, billing, API keys, or infrastructure access.

Please do not include real customer data, third-party secrets, or destructive payloads.

## Response expectations

Paravane Labs will review reports as promptly as possible and may follow up for clarification. Remediation timing depends on severity, exploitability, affected systems, and release risk.

## Scope

This policy applies to Paravane Labs public repositories and public-facing Paravane Labs systems. Individual repositories may define more specific supported versions, disclosure expectations, or reporting paths.
