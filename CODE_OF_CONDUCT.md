# Code of Conduct

Paravane Labs public repositories are intended to be practical, respectful spaces for discussing software, documentation, security, and product integration work.

## Expected behavior

Participants are expected to:

- Be respectful and constructive.
- Assume technical discussions should remain focused on the work.
- Give useful context when opening issues or pull requests.
- Accept maintainers' decisions about project scope, priorities, and release timing.
- Avoid sharing secrets, credentials, private customer data, or sensitive operational details in public issues or pull requests.

## Unacceptable behavior

Unacceptable behavior includes:

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing another person's private information without permission.
- Repeatedly derailing technical discussions.
- Spam, scams, malware, credential harvesting, or abusive automation.
- Publicly disclosing security vulnerabilities before Paravane Labs has had a reasonable chance to review and remediate them.

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, block users, or take other moderation actions when behavior conflicts with this Code of Conduct.

Security concerns should be reported through the process in `SECURITY.md`, not through public issues.
